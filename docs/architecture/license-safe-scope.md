# RPGMOD 라이선스 안전 구현 범위

## 1) 원칙
- 외부 플러그인/모드의 코드를 직접 복사하지 않는다.
- 기능 아이디어, UX 흐름, 설정 철학만 참고하고 내부 코드로 재구현한다.
- 모듈별로 "참고 출처", "재구현 포인트", "복제 금지 요소"를 분리 기록한다.

## 2) 허용 범위
- 명령어 체계, 노드 네이밍 패턴, 설정 구조 아이디어 참조
- 밸런싱 방식(배율/곡선), 운영 도구 개념(리로드/히스토리/undo) 참조
- 툴팁/GUI 정보 배치 방식 참조

## 3) 금지 범위
- 외부 프로젝트의 클래스/메서드 구현 그대로 복붙
- 외부 문서의 대량 문구 복사(설명/메시지/주석 포함)
- 라이선스 조건 미검증 상태에서 코드 혼합

## 4) 모듈별 체크리스트
### permission-core
- 노드/그룹/컨텍스트/임시권한 구조는 자체 JSON 모델 사용
- 평가 로직과 캐시는 자체 구현

### progression-core
- 레벨/전직/숙련도 계산은 자체 수식 서비스 사용
- `progression-formulas.json` 으로 실시간 수치 조정

### itemization-core
- affix/socket 롤 규칙은 `weapons.json` 기반 자체 구현
- 아이템 롤 결과 저장 포맷은 `rpgmod_weapon_roll` 키 사용

## 5) 운영 수칙
- 변경 전후 `compileJava` 검증 필수
- 신규 시스템은 먼저 모듈로 분리 후 기존 흐름에 연결
- 대형 확장 시 DB/네트워크 분리 전까지는 JSON + 캐시 구조 유지
